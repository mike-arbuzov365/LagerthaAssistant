## Що таке вайб-дизайн

Вайб-дизайн — це підхід, де AI (Codex, Claude Code) є основним виконавцем дизайнерських задач, а людина — стратегом і ревьюером. Принципи ті самі що й у вайб-кодингу:

- Маленькі кроки з постійною верифікацією
- Один компонент або екран за раз
- Завжди перевіряти на реальному пристрої (не тільки в Figma)
- AI готує — людина ревьюює і апрувить

## Процес: від ідеї до Figma

```
1. Отримати завдання
   └── "Треба компонент Integration Card"

2. Прочитати 08-design-system.md
   └── Знайти відповідні токени, перевірити чи є схожий компонент

3. Описати специфікацію (офлайн)
   └── Розміри, кольори, стани, поведінка

4. Написати JSX-код компоненту
   └── Код для React / TMA

5. Додати запис у figma-queue.md
   └── Формат QUEUE-NNN з повною специфікацією

6. НЕ підключатись до Figma
   └── Продовжити наступне завдання

7. Накопичити >= 5 змін АБО milestone
   └── Запросити явний апрув власника

8. Тільки після явного апруву підключитись один раз
   └── Одне підключення → застосувати всю чергу
```

## Правила для AI (Codex / Claude Code)

### Перед початком будь-якого дизайн-завдання

```
1. Прочитати docs/08-design-system.md
2. Прочитати docs/figma-queue.md (що вже є в черзі)
3. НЕ підключатись до Figma MCP без явної інструкції
4. Навіть якщо в черзі >= 5 pending, без апруву не підключатись
5. Всі зміни — спочатку в queue
```

### При описі компонента

```
- Використовувати тільки токени з 08-design-system.md (не хардкодити hex)
- Описувати ВСІ стани (default, hover, active, error, disabled, loading)
- Давати точні розміри в px
- Включати JSX reference code
- Вказувати TMA Safe Areas якщо екранний компонент
```

### При підключенні до Figma MCP

```
- Тільки якщо є явний апрув власника І в figma-queue.md є `>= 5` pending АБО milestone (hard gate, без винятків)
- Застосувати ВСЮ чергу за одну сесію
- Почати з Variables → Styles → Atoms → Molecules → Screens
- Позначити всі applied записи в queue
- Записати дату застосування
```

---

## Заборонено

- Підключатись до Figma MCP для однієї зміни
- Підключатись до Figma MCP без апруву власника (навіть при >= 5 pending)
- Хардкодити кольори (завжди через Variable)
- Ігнорувати TMA Safe Areas
- Робити інтерактивну зону елементів менше 44px по висоті (мінімум для touch)
- Використовувати інтерактивний/операційний текст менше 13px
- Забувати стан disabled та loading

## Тестування дизайну

**До підключення до Figma:**
- Перевірити JSX код у браузері (localhost)
- Перевірити на мобільних розмірах (375px, 390px)

**Після підключення до Figma:**
- Відкрити в Figma Mirror на реальному телефоні
- Перевірити всі стани компонента (Component Properties)
- Перевірити TMA режим (якщо є Telegram девтулс)
